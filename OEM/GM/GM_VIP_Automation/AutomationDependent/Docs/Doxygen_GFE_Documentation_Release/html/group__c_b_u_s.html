Function Wait and check if a physical signal value is not changing his value during a given time.

Parameters
[in]aSignalName- Signal name from the database.
[in]aTimeToWaitForSignal- The given time that the tester expects that the signal value is not changed
[in]aSignalValue- the expected signal value
[in]aBusContext- the bus name context
[out]aMappedResponseInfo- the information which is going to be printed in the report
Returns
0 (NotOk) - if the signal values are changed
1 (Ok) - if the signal values are not changed
-1 (Error) - error (the bus context name is wrong / the time to wait for signal is zero / the signal does not have the expected value)
Note
  • An timeout must be set before this step

Function Wait and check if a message is send with his values inside of a given range or not. Function is used for raw values.

Parameters
[in]aSignalName- Signal name from the database.
[in]aTimeToWaitForSignal- Maximum time to be waited (if the value is 0 the function should get the first value of the signal)
[in]aMsgCycleTime- The message cycle time (from database)
[in]aMinValue- The minimum of the signal.
[in]aMaxValue- The maximum of the signal.
[in]aBusContext- the bus name context
[out]aMappedResponseInfo- the information which is going to be printed in the report
Returns
0(NotOk) - if the signal values are not in range
1(Ok) - if the signal values are in range
-1(Error) - an error occured (Message cycle time is zero / lower limit is bigger than higher limit / timeout provided is smaller than message cycle time / bus cotext name is wrong)
