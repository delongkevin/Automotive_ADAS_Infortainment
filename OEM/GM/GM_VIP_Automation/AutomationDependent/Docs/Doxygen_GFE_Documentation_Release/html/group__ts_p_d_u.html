Function that executes the action to wait an amount of time for the specified Autosar PDU Name and store the payload data in the output byte array.

Parameters
[in]aPDUNameFromDB- Name of the Autosar PDU from the arxml database.
[in]aArrayToStorePayloadData- The byte array in which the payload data is copied.
[in]aWaitingTime- Maximum wait time, in miliseconds, to check if the Autosar PDU arrives or not (cannot be < 1).
[in]aBusContext- The bus context.
Returns
None
Note
  • The aPDUNameFromDB parameter should be dragged/dropped from arxml database listed in CAPL Network Symbols window.
    Check the example in the test file: tsPDU_Tests.can for more details.

[in]aWaitingTime- Maximum wait time, in miliseconds, to check if the Autosar PDU arrives or not.

Function that checks if an Autosar PDU cycle time is in a given range [aPDUCycleTimeMS - (aPDUCycleTimeMS * aPermittedTimeDiffPerCent/100), aPDUCycleTimeMS + (aPDUCycleTimeMS * aPermittedTimeDiffPerCent/100)] or not for a given number sample times.

Parameters
[in]aPDUNameFromDB- Name of the Autosar PDU from the arxml database.
[in]aNumberOfSampleTimes- The number of samples time to test that the cycle time of Autosar PDU is in range or not
[in]aPDUCycleTimeMS- The Autosar PDU cycle time from data base (in milliseconds)
[in]aPermittedTimeDiffPerCent- The permitted time difference of the upper and lower limit of the cycle time range (per cent).
[in]aBusContext- The bus context.
Returns
no return
Note
  • The aPDUNameFromDB parameter should be dragged/dropped from arxml database listed in CAPL Network Symbols window.
    Check the example in the test file: tsPDU_Tests.can for more details.'
  • Note
    • Example: if a Autosar PDU has a cycle time of 100 milliseconds and the procent is 15. The calculated lower accepted
      value for cycle time is 85ms and upper accepted value for cycle time is 115 ms.
