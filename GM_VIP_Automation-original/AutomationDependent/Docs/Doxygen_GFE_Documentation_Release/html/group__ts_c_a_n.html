Function that executes the action to save in a CSV file, the specified number of log entries from the already populated CAN Message log structure.

Parameters

[in]aLogDataToSaveInCSV- Structure from where the specified CAN message log entries are extracted and written in the CSV file (has to be already populated using: A_CAN_StartLoggingMessages() and A_CAN_StopLoggingMessages()).
[in]aNrOfLogEntriesToWriteInCSV- Number of log entries to be written in the CSV file (cannot be 0 or larger than the total size of the array structure: aLogDataToSaveInCSV).
Returns
None

  1. A_CAN_StartLoggingMessages(dwLogSlotIDsArray_3Elem, ChannelNr, NrOfLogEntries);

void A_CAN_StartLoggingMessages ( dword  aArrayMessageIDs[],
byte  aChannelNumber,
int64  aNumberOfLogEntries 
)

Function that executes the action to enable the CAN Message Logging functionality, based on a specified filter (MsgID's and ChannelNr).

Parameters
[in]aArrayMessageIDs- Array with the MessageID's which are going to be logged (if first element of the array is 0, all messages are logged).
[in]aChannelNumber- The Channel/Cluster number where the message(s) is expected to arrive (if 0, it will log every occurrence of the MsgID, regardless of the cluster it arrives).
[in]aNumberOfLogEntries- The maximum number of how many message(s) log entries is expected to arrive ( > 0 and recommended to not go over 500.000 as it could influence the environment and memory allocation).
Returns
None
Note
  • After enabling the logging functionality it is necessary to use: testWaitForTimeout(WaitTimeInMs) - only within this time the frames are added in the buffer structure.
  • Even though dynamic memory allocation is used for the buffer structure array, a limit of maximum: 500.000 entries should suffice in order to avoid excessive memory allocation.
  • Example of correct function call sequence:
  1. A_CAN_StartLoggingMessages(dwLogSlotIDsArray_3Elem, ChannelNr, NrOfLogEntries);
  2. testWaitForTimeout(5000); // Only during this time, the log is active. Expected data which arrives within 5 seconds, is saved in the back-end buffer.
  3. A_CAN_StopLoggingMessages(strSaveLogData_1500Elem, i64NumberOfExtractedLogEntries);
  4. A_CAN_SaveLogDataToCSVFile(chAbsPathOf_CSVLogFile, strSaveLogData_1500Elem, i64NumberOfLogEntriesToSave);

Parameters

[in]aTimeout- max wait time for the message.
[in]aMinAbsCycleTime- maximum time for message cycle.
[in]aMaxAbsCycleTime- minimum time for message cycle.
[in]aBusContext- the specific bus context
Returns
no return
Note
  • Event is generated if the time between sends of message is smaller than aMinAbsCycleTime and aMaxAbsCycleTime (example: if Message cycle
    time is 50ms and difference between aMinAbsCycleTime and aMaxAbsCycleTime is 60ms the event is not generated)
  • If the limits (aMinAbsCycleTime or/and aMaxAbsCycleTime) are zero, this function does not check those limits.
