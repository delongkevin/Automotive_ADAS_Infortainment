int iFctn_CAN_CycleTimeCheckByMessageID ( dword  aMsgID,
dword  aWaitingTime,
float  aMinCycleTime,
float  aMaxCycleTime,
char  aBusContext[],
char  aCycleTimeCheckInfo[] 
)

Function verify the cyclicality of a message between the entered limits.

Parameters
[in]aMsgID- ID of message
[in]aWaitingTime- Time duration of a check
[in]aMinCycleTime- Maximum time for message cycle.
[in]aMaxCycleTime- Minimum time for message cycle.
[in]aBusContext- The specific bus context
[out]aCycleTimeCheckInfo- Extra information regarding the function output (displayed in the test report).
Returns
0 (NotOk) - if message cycle time is violated.
1 (Ok) - if message is in cycle time.
-1 (Error) - if cycle time check could not be done / name of the bus context is wrong / the used value timeout or limits are wrong (too small)
Note
  • Event is generated if the time between sends of message is smaller than aMinAbsCycleTime and aMaxAbsCycleTime (example: if Message cycle
    time is 50ms and difference between aMinAbsCycleTime and aMaxAbsCycleTime is 60ms the event is not generated)
  • If the limits (aMinAbsCycleTime or/and aMaxAbsCycleTime) are zero, this function does not check those limits.

◆ iFctn_CAN_StartLoggingMessages()

int iFctn_CAN_StartLoggingMessages ( dword  aArrayMessageIDs[],
byte  aChannelNumber,
int64  aNumberOfLogEntries,
char  aStartLoggingMsgInfo[] 
)

Function that enables the CAN Message Logging functionality, based on a specified filter (MsgID's and ChannelNr).

Parameters
[in]aArrayMessageIDs- Array with the Message ID's which are going to be logged (if first element of the array is 0, all messages are logged).
[in]aChannelNumber- The channel/cluster number where the message(s) is expected to arrive (if 0, it will log every occurrence of the MsgID, regardless of the cluster it arrives).
[in]aNumberOfLogEntries- The maximum number of how many message(s) log entries is expected to arrive ( > 0 and recommended to not go over 500.000 as it could influence the environment and memory allocation).
[out]aStartLoggingMsgInfo- Extra information regarding the function output (displayed in the test report).
Returns
0 - Invalid MsgIDs within the array.
1 - Successfully enabled the logging mechanism and valid parameters were provided.
-1 - Error, could not enable the logging mechanism (already enabled or expected log entries is 0) or another error occurred.
Note
  • Even though dynamic memory allocation is used for the buffer structure array, a limit of maximum: 500.000 entries should suffice in order to avoid excessive memory allocation.
