Function that executes the action to save in a CSV file, the specified number of log entries from the already populated log structure.

Parameters
[in]aAbsoluteCSVFilePath- The absolute path to the CSV file. Only .csv extension allowed (Example: C:\UserName\Desktop\MyLogFile.csv).
[in]aLogDataToSaveInCSV- Structure from where the specified log entries are extracted and written in the CSV file (has to be already populated using: A_FR_StartLoggingFrames() and A_FR_StopLoggingFrames()).
[in]aNrOfLogEntries- Number of log entries to be written in the CSV file (cannot be 0 or larger than the total size of the array structure: aLogDataToSaveInCSV).
Returns
None
Note
  • Recommended to open the saved CSV file with Excel (make use of WordWrap in case of large payload data).
  • Example of correct function call sequence:
  1. A_FR_StartLoggingFrames(wLogSlotIDsArray_4Elem, ClusterNr, ChannelAB, BaseCycle, CycleRep, NrOfLogEntries);
  2. testWaitForTimeout(5000); // Only during this time, the log is active. Expected data which arrives within 5 seconds, is saved in the back-end buffer.
  3. A_FR_StopLoggingFrames(strSaveLogData_1500Elem, i64NumberOfExtractedLogEntries);
  4. A_FR_SaveLogDataToCSVFile(chAbsPathOf_CSVLogFile, strSaveLogData_1500Elem, i64NumberOfLogEntriesToSave);

Function that executes the action to send a dynamic FlexRay frame with the desired payload data, on the specified cluster and channel(A,B).

Parameters
[in]aFrameName- The dynamic frame which is going to be sent.
[in]aPayloadToBeSent- Byte array with the desired payload to be sent together with the dynamic frame (payload length is determined from the size of the array, which should not be greater than 254 bytes).
[in]aFlexRayCluster- The cluster of the FlexRay protocol. Ex: 1(CH_Flexray), 2(CH_Flexray2).
[in]aChannel- The channel of the flexray (Ex: A or B).
Returns
None
Note
  • If the byte length of the array is uneven(ex: 5, 7), the payload length is rounded automatically to the next even value (ex: 6, 8)
    Due to possible configuration issues, channel A+B is not allowed for the current function (you can send it individually though).

◆ E_FR_CycleTimeCheckByFrameName()

void E_FR_CycleTimeCheckByFrameName ( dbFrFrame *  aFrameNameFromDB,
dword  aWaitingTime,
float  aMinCycleTime,
float  aMaxCycleTime 
)

Function that evaluates for a period of time if the cycle time of a frame is within the expected min and max cycle time.

Parameters
[in]aFrameNameFromDB- Name of the FlexRay Frame from the database.
[in]aWaitingTime- Maximum wait time, in miliseconds, to evaluate the cycle time of the specified FR Frame.
[in]aMinCycleTime- Minimum expected cycle time, in miliseconds.
[in]aMaxCycleTime- Maximum expected cycle time, in miliseconds.
Returns
None
Note
  • Event is generated if the time between sends of the frame is smaller than aMinCycleTime and aMaxCycleTime (Ex: if Frame cycle
    time is 50ms and difference between aMinCycleTime and aMaxCycleTime is 60ms the event is not generated).
  • If one of the limit (aMinCycleTime or aMaxCycleTime) is zero, the function does not check that particular limit (not allowed for both to be 0 at the same time).
  • The aFrameNameFromDB parameter can be called in following way: [Data Base Name] :: [Data Base Node] :: Frame Name
    (example: CH_FlexRay(Data Base Name - optional) :: Tester(Data Base Node-optional) :: Test_frame1(Frame Name).
  • The optional attributes (specified on "[", "]") must be used if frames with the same name are present on different data bases/FR nodes.
