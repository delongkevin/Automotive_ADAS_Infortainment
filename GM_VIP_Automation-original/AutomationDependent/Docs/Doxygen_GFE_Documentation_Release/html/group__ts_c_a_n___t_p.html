◆ A_CAN_TP_WaitForAckFrame()

void A_CAN_TP_WaitForAckFrame ( char  aNodeName[],
char  aNodeClusterName[],
dword  aWaitingTime 
)

Function that executes the action to wait an amount of time, for an Acknowledge Frame(3F) to be received on a certain node.

Parameters
[in]aNodeName- Name of the node where the Ack Frame is expected to be received.
[in]aNodeClusterName- Name of the cluster where the node is present.
[in]aWaitingTime- Maximum wait time, in miliseconds, to wait for arrival of Ack Frame (cannot be < 1ms).
Returns
None
Note
  • Acknowledge Frame is confirmed to the sender node after the data has been received by the destination node (accessing/waiting for the data is no longer possible in this case as it already arrived on the node).
  • ISO TP specifies an unacknowledged data transfer, i.e. the sender does not get any confirmation that the receiver has received all of the data correctly. An extension to this standard behavior can be activated with these functions – please note that all communication partners have to implement this extension!
