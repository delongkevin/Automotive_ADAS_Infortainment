char  aWaitFrameInfo[] 
)

Function that evaluates if a FlexRay Frame arrives, on a certain FR Cluster and its associated channel, within the specified wait time (based on SlotID, BaseCycle and CycleRepetition).

Parameters
[in]aFrameSlotID- The Slot ID of the expected frame (range allowed: 1 - 2047).
[in]aFrameBaseCycle- Numeric base cycle. Must be less than the repetition factor (range allowed: 0 - 63).
[in]aFrameCycleRepetition- Cycle repetition factor. Value has to be a power of 2 (1, 2, 4, 8, 16, 32 or 64).
[in]aChannel- The channel(s) of the specified cluster (ChannelA = 1, ChannelB = 2, ChannelAB = 3).
[in]aWaitingTime- Maximum wait time, in miliseconds, to check if the frame arrives.
[in]aExpectedFramePresence-The expected presence of the FR Frame: Present or Absent.
[in]aFlexRayClusterName- The exact name of the FlexRay cluster (Ex: CH_FlexRay, CH_FlexRay2). Necessary in order to set the bus context.
[out]aWaitFrameInfo- Extra information regarding the function output (displayed in the test report).
Returns
0 - Unexpected presence of the frame (present instead of absent / absent instead of present).
1 - The presence of the frame is as expected.
-1 - Parameters are invalid, sub-string conversion failed, incorrect cluster or another error occurred.

Function that checks if a FlexRay Frame is received in a specific time or not, with the expected Equal/NotEqual payload data.

Parameters
[in]aFrameNameFromDB- Name of the FlexRay Frame from the arxml database.
[in]aExpectedPayloadData- Byte array with the exact size and desired content of what the received payload should be identical/different (depending on the aExpectedCompareAction).
[in]aExpectedCompareAction- The compare action between the provided array and received payload: Equal or NotEqual.
[in]aWaitingTime- Maximum wait time, in miliseconds, to check if the FR Frame arrives or not.
[out]aFrameDataCompareInfo- Extra information regarding the function output (displayed in the test report).
Returns
0 - Unexpected content arrived, frame did not arrive at all or the size of the array is not correct.
1 - The payload data which arrived is as expected (based on aExpectedCompareAction: 0-Equal/1-NotEqual).
-1 - Conversion has failed or another error occurred (check the Error Code).
